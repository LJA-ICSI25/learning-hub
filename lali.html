Computer Hardware

Every computer is built from the same core components. Understanding what each part does — and how they interact — is fundamental to the Tech+ exam.

⚡The IPO Model — How Computers Work

All computers follow the Input → Processing → Output model, with Storage as an additional function.

Input

Data enters the computer. Examples: keyboard, mouse, microphone, barcode scanner, webcam, touchscreen.

Processing

The CPU performs calculations. Speed is measured in GHz (gigahertz = billions of cycles/second).

Output

Results are presented to the user. Examples: monitor, printer, speakers, projector.

Storage

Data is saved for future use. Can be volatile (RAM — lost when powered off) or non-volatile (HDD, SSD — persistent).

🧠CPU — Central Processing Unit

The CPU is the "brain" of the computer. It executes all instructions and performs calculations. Key specs:

  • Clock Speed (GHz) — How many instruction cycles per second. Higher = faster processing.
  • Cores — Multiple cores allow parallel processing (dual-core, quad-core, etc.)
  • Cache — Ultra-fast memory built into the CPU for frequently accessed data.
  • Architecture — x86 (Intel/AMD desktop), ARM (mobile devices).
📝

Exam Scenario

A video editor's workstation is slow at rendering. To fix this, upgrade the CPU (measured in GHz). For graphics-heavy tasks, upgrade the GPU.

🎮GPU — Graphics Processing Unit

Handles math-heavy parallel calculations for rendering graphics and video. Modern GPUs are also used for AI/ML workloads. Distinct from CPU which handles general sequential tasks.

🔌PSU — Power Supply Unit

Converts AC power from the wall into DC power for internal components. High-end GPUs require PSUs with adequate wattage. Failure can damage all components.

🗂️Motherboard — The Central Hub

The motherboard is the large printed circuit board (PCB) that everything plugs into. It provides the electrical connections and "bus" lines that allow all components to communicate.

Input Devices

Keyboard, mouse, microphone, touchscreen, barcode scanner, webcam, scanner, joystick, drawing tablet

Output Devices

Monitor, printer, speakers, projector, headphones, haptic feedback

⚠️

Trick Question Alert

A barcode scanner is an input device — it sends data into the computer. A printer is an output device — it produces physical output from the computer.

Number Systems & Data Units

Computers use different number systems to represent data. Understanding how bits and bytes scale up is critical for hardware and networking questions.

🔢The Four Number Systems

SystemBaseDigits UsedExampleUsed For
BinaryBase-20, 11010CPU communication, direct hardware
DecimalBase-100–942Human everyday math
HexadecimalBase-160–9, A–FA3F5MAC addresses, color codes, memory
OctalBase-80–7755Linux file permissions
💡

Quick Trick

If you see letters A–F mixed with numbers, it's hexadecimal. If it's only 0s and 1s, it's binary. MAC addresses like 00:25:9C:4D:5A:6B are always hexadecimal.

📊Data Units — Smallest to Largest

Memory and storage are measured in multiples. Memorize this sequence:

1 Bit → ×8 → 1 Byte (8 bits) → ×1024 → Kilobyte (KB) → Megabyte (MB) → Gigabyte (GB) → Terabyte (TB) → Petabyte (PB)

Storage Capacity

Consumer hard drives are measured in terabytes (TB). Enterprise data centers may use petabytes. Modern HDDs: 1–20 TB. SSDs: 256 GB – 4 TB.

Network Speed

Data transfer speeds use bits (not bytes): Kbps → Mbps → Gbps. Enterprise networks use Gbps. Fiber optic often measured in Gbps.

Storage & Memory

Understanding the difference between volatile and non-volatile storage, and the various storage technologies, is essential.

⚡Volatile vs. Non-Volatile Storage

Volatile (Temporary)

Data is LOST when power is removed.

Examples: RAM (Random Access Memory). Used for currently running programs. Fast access, temporary. The OS uses RAM to track which applications are using system memory (memory management).

Non-Volatile (Persistent)

Data is KEPT even without power.

Examples: HDD, SSD, Flash drives, ROM, optical discs. Used for long-term data storage. Operating systems, files, and programs are stored here.

⚠️

Key Exam Fact

RAM is volatile. ROM is non-volatile. NVMe, SSD, HDD, and flash drives are all non-volatile. If a question asks "which storage type is volatile?" — the answer is always RAM.

🔑BIOS / UEFI — The First Software to Load

When a computer powers on, the BIOS (Basic Input/Output System) or its modern replacement UEFI loads first — before the operating system. It is firmware (software embedded in a chip on the motherboard).

  • POST (Power-On Self-Test) — BIOS checks all hardware is functioning. Warning beeps = hardware failure.
  • Boot order — BIOS/UEFI determines which device to boot from (HDD, SSD, USB, etc.)
  • CMOS Battery — Keeps BIOS settings and the system clock alive when the computer is unplugged.

🛡️Firmware — Embedded Software

Firmware is software that is permanently embedded into a hardware chip (ROM). It controls low-level device operation. Examples:

  • BIOS/UEFI on motherboard
  • Firmware on a GPU (graphics microchip)
  • Embedded OS in smart TVs, gaming consoles, refrigerators
  • Router firmware
📝

Firmware vs. Drivers vs. Software

Firmware = embedded in chip, controls hardware itself. Drivers = OS software that lets the OS talk to hardware. Software = applications users run.

🔑MAC Address vs. IP Address

MAC Address

A hardware address burned into the NIC at manufacturing. Cannot be changed (permanent/unique). Used at Layer 2. Format: 00:25:9C:4D:5A:6B (hexadecimal). Switches use MAC addresses to direct traffic within a local network.

IP Address

A logical address assigned by the network. Can be changed (dynamic or static). Used at Layer 3. Format: 192.168.1.100 (IPv4). Routers use IP addresses to route traffic between networks.

🏠Network Types

  • LAN (Local Area Network) — Small area: office, home. Devices connected by switch/router.
  • WAN (Wide Area Network) — Large geographic area. The Internet is the largest WAN.
  • PAN (Personal Area Network) — Bluetooth, NFC. Very short range.
  • MAN (Metropolitan Area Network) — City-wide network.

🌐Internet Connection Types

  • DSL (Digital Subscriber Line) — Over phone lines. RJ-11 connector.
  • Cable — Over coaxial cable lines. Fast, shared with neighbors.
  • Fiber Optic — Light through glass fiber. Fastest. Gbps speeds.
  • Satellite — Wireless from space. High latency.
  • Cellular — 4G/5G mobile network.

🛠️Connectivity Tools & Commands

Ping

Tests basic connectivity between two devices. Sends ICMP packets and waits for a response. Use: ping 192.168.1.1

Tracert / Traceroute

Traces the path data takes across a network, showing each hop (router) along the way. Identifies where delays occur.

DHCP

Dynamic Host Configuration Protocol — automatically assigns IP addresses to devices on the network.

Short-Range Wireless Technologies

📲Bluetooth

Short-range wireless for device pairing. Range: ~10 meters. Used for: headphones, keyboards, speakers, file transfer between nearby devices.

💳NFC

Near Field Communication. Extremely short range (~4 cm). Used for: contactless payments (tap-to-pay), access cards, pairing. Examples: Apple Pay, Google Pay, credit card tap terminals.

⚡Thunderbolt

High-speed wired/port technology (not strictly wireless). Supports data, video, and power over one cable. Bidirectional. Very fast (up to 40 Gbps). Common on MacBooks.

🏠Smart Home & IoT Devices

IoT (Internet of Things) devices are everyday objects connected to the internet.

Home Assistant

Voice-activated smart home control device. Example: Amazon Echo, Google Home. Used for voice commands to control smart devices.

IoT Examples

Smart refrigerator, video doorbell, smart thermostat, IP security cameras, smart light bulbs. These all have embedded OSes.

🖥️Display Technologies

Casting

Sends content to a TV/display while the source device can do other things. Example: Chromecast, AirPlay. The source offloads video processing to the TV.

Mirroring

The display shows an exact copy of the source screen. Both devices show the same thing. The source device does all processing.

Operating Systems

The OS is the software layer between the hardware and user applications. Understanding OS types, functions, and interfaces is heavily tested.

⚙️What an OS Does — Core Functions

User Interface

GUI (Graphical User Interface) uses windows and icons. CLI (Command Line Interface) uses text commands — powerful but harder to learn syntax.

Process Management

The OS runs, tracks, and schedules all running programs. A process = an instance of a running program. Services = background programs.

Memory Management

The OS tracks which applications are using RAM. When 20 browser tabs open and the system slows — that's memory management struggling.

Disk Management

The OS manages reading/writing to storage. The file system (NTFS, FAT32, ext4) organizes data on disk.

Access Control

Controls which users can access which files/resources. Managed through user accounts and permissions (ACL — Access Control List).

Driver Management

Loads and manages device drivers so hardware can communicate with the OS. Plug and Play = OS automatically installs drivers for new hardware.

⚠️

What is NOT an OS Function?

"Connecting to the internet" is NOT a core OS function — that's a network service/application layer responsibility. The OS provides the infrastructure but connecting to the internet involves your browser, NIC drivers, and network settings.

🗂️File Management

Windows: File Explorer

The GUI tool for browsing and managing files in Windows. Also useful for copying files to USB drives.

macOS: Finder

macOS equivalent of File Explorer. Used to navigate and manage files and folders on Mac.

File Extensions

Suffixes like .docx, .html, .mp4 that identify file types. The OS uses these to determine which app opens the file.

File Attributes

Properties including: file name, creation date, size, read-only flag, hidden flag, system file flag, who can access/edit/encrypt the file.

Access Control List (ACL)

A list attached to files/folders defining which users can read, write, execute, or delete. This is how the OS enforces file permissions.

🌐Browser Features You Need to Know

Private / Incognito Mode

Browsing history, cookies, and temporary files are NOT saved locally after the session. Useful on shared/public computers.

Browser Cache

Stores copies of web pages for faster loading. If you see an old version of a site — clear the cache.

Profile Synchronization

Syncs bookmarks, passwords, history across multiple devices using the same browser account.

SSL/TLS Certificates

Validates a website's identity and enables encrypted HTTPS connections. An IT manager checks for valid certificates to confirm security.

🔄Initial OS Setup — Localization

When first setting up an OS, besides the computer name, you must configure localization settings: language, region, date/time format, currency, and keyboard layout. This ensures the OS behaves correctly for your geographic location.

// Domain 3.3

Cloud Computing & Virtualization

Cloud and virtualization are modern infrastructure concepts. Understand the service models and deployment types.

☁️Cloud Service Models

IaaS — Infrastructure as a Service

Cloud provider gives you raw computing resources: servers, networking, storage. You manage the OS and software. Most control, most responsibility. Example: AWS EC2, Azure VMs.

PaaS — Platform as a Service

Cloud provides the platform and runtime to deploy applications. You just write and deploy code. Example: Heroku, Google App Engine.

SaaS — Software as a Service

Fully managed software accessed via web browser. Cloud provider handles everything. You just use the app. Examples: Gmail, Salesforce, Microsoft 365, Zoom.

🏗️Cloud Deployment Models

Public Cloud

Infrastructure owned and managed by a third-party provider, shared with other customers. Examples: AWS, Azure, Google Cloud.

Private Cloud

Infrastructure dedicated to a single organization. Can be on-premises or hosted. More control and security.

Hybrid Cloud

Combines on-premises infrastructure with cloud resources. Organizations use local servers for sensitive data and cloud for scalable workloads.

Multi-Cloud

Using services from multiple cloud providers simultaneously (e.g., AWS + Azure). Avoids vendor lock-in.

🖥️Virtualization

Virtualization allows multiple virtual machines (VMs) to run on one physical computer simultaneously. A hypervisor manages the VMs.

Type 1 Hypervisor (Bare Metal)

Installed directly on the physical hardware — no host OS. Most efficient for enterprise. Examples: VMware ESXi, Microsoft Hyper-V, Citrix XenServer.

Type 2 Hypervisor (Hosted)

Runs on top of an existing OS. Less efficient but easy to use on personal computers. Examples: VMware Workstation, VirtualBox, Parallels.

📝

Exam Scenario

Inga wants to run both Windows and Linux on one desktop simultaneously — the technology is Virtualization. A Type 2 hypervisor (like VirtualBox) would be installed on her existing OS.

☁️Cloud Storage & Backup

Cloud Advantages

Remote access from anywhere, automatic backups, disaster recovery, offsite storage, scalable. Fulfills the "offsite backup" requirement for business continuity.

Cloud Disadvantages

Requires reliable internet connection to access/backup data. Privacy concerns (data on third-party servers). Ongoing subscription cost.

💡

Backups — Key Rule

Always verify backups by testing restoration procedures. A backup that can't be restored is useless. Testing restoration is the most critical step after creating a backup.

Cybersecurity

Security is one of the most heavily weighted domains. Understand the CIA Triad, types of attacks, authentication, and security controls.

🔺The CIA Triad — Foundation of Security

C — Confidentiality

Only authorized people can access sensitive data. Protected by: encryption, access controls, MFA, password policies. Example: encrypting a hard drive.

I — Integrity

Data is accurate and unaltered. Protected by: hashing, digital signatures, checksums. Ensures data hasn't been tampered with during storage or transmission.

A — Availability

Systems and data are accessible when needed. Protected by: redundancy, backups, UPS, RAID, failover systems. Prevents denial of service.

🎭Social Engineering Attacks

Social engineering manipulates people (not systems) into revealing confidential information or taking harmful actions. "Hacking humans, not computers."

Phishing

Fake email that looks legitimate (bank, employer) asking you to click a link and enter credentials. Most common social engineering attack.

Spear Phishing

Targeted phishing aimed at a specific person or organization. More convincing because it's personalized.

Tailgating / Piggybacking

Following an authorized person through a secure door without using your own credentials. Physical security threat.

Vishing

Voice phishing — using phone calls to trick victims into revealing information. "This is the IRS calling..."

🔐Authentication & Access Control

MFA / 2FA

Multi-Factor Authentication requires 2+ forms of evidence: something you know (password), something you have (phone app), something you are (fingerprint/retina).

Principle of Least Privilege

Give users only the access they need to do their job — nothing more. Limits damage if account is compromised.

Single Sign-On (SSO)

One login grants access to multiple systems. Convenient but risky — if compromised, all systems are exposed.

Non-Repudiation

Proof that an action occurred and was performed by a specific person. Methods: digital signatures, biometrics, audit logs, surveillance cameras.

Role-Based Access Control

Access is based on job role, not individual. Example: "Backup Administrators" can access all servers.

Password Expiration

Forces periodic password changes to limit the time window a stolen password can be used by an attacker.

🛡️Security Controls

Preventive Controls

Stop incidents before they happen. Examples: firewall, MFA, encryption, strong password policy, security awareness training.

Detective Controls

Identify incidents while or after they occur. Examples: antivirus (scanning), security cameras, audit logs, intrusion detection systems (IDS).

Corrective Controls

Fix damage after an incident. Examples: restoring backups after ransomware, incident response, system reimaging.

Physical Controls

Physical barriers: badge readers, door locks, cable locks, security guards, surveillance cameras. Protect physical access to hardware.

Technical/Logical Controls

Software/digital protections: firewalls, antivirus, encryption, ACLs, MFA, VPN. Protect digital access and data.

🔒Encryption & Protocols

HTTPS

HTTP secured by SSL/TLS encryption. Protects web traffic in transit. Look for the padlock icon. The certificate validates the site's identity.

VPN

Virtual Private Network. Creates encrypted tunnel over the internet. Used for secure remote access and public Wi-Fi protection.

Full Disk Encryption

Encrypts entire hard drive. If laptop is stolen, data is unreadable without the encryption key. BitLocker (Windows), FileVault (macOS).

File-Level Encryption

Encrypts individual files or folders rather than the entire disk. More granular control.

🔥Firewall

A firewall inspects incoming and outgoing network traffic against a set of rules, blocking unauthorized access. It's placed at the edge of the network (between internal LAN and the internet).

  • Packet filtering — Examines packet headers (IP, port) and allows/denies based on rules.
  • Stateful inspection — Tracks connection state, more sophisticated than simple packet filtering.
  • Application-layer firewall — Inspects actual content of traffic (Layer 7).

🔑Relational Database Key Concepts

Primary Key

A field that uniquely identifies each record in a table. Must be unique, cannot be null. This is the "one" side in one-to-many relationships. Good: Employee ID, auto-increment number. Bad: First name (not unique).

Foreign Key

A field in one table that references the primary key of another table. This creates the relationship (link) between tables. Enables automatic updates across related records.

Schema

The blueprint/structure of the database — defines tables, fields, data types, and relationships. The schema describes the database before data is entered.

Query

A command to "ask" the database for specific data. SQL SELECT statements are queries. Example: "Find all employees who've worked here more than 5 years."

Constraint

A rule that enforces data integrity. Example: NOT NULL constraint prevents fields from being left empty. UNIQUE constraint ensures no duplicate values.

Tables, Fields, Records

Tables = the data container. Fields (columns) = categories of data. Records (rows) = individual entries. The table organizes data in rows and columns.

💾SQL — Structured Query Language

SQL is the standard language used to communicate with relational databases (MySQL, SQL Server, PostgreSQL, etc.).

-- Retrieve specific data from a table SELECT first_name, last_name, hire_date FROM employees WHERE years_worked > 5 ORDER BY last_name; -- Link two tables using a JOIN (using foreign key) SELECT orders.order_id, customers.name, orders.amount FROM orders INNER JOIN customers ON orders.customer_id = customers.id; -- Insert a new record INSERT INTO employees (name, department, salary) VALUES ('Jane Smith', 'IT', 75000);

📊Data Concepts

Metadata

"Data about data." Describes a file's properties: creation date, file size, author, format. Example: a photo's metadata includes when/where it was taken.

Big Data

Characterized by 3 V's: Volume (extremely large), Velocity (fast-moving), Variety (complex/varied types). Requires special processing tools.

Data Analytics

The process of examining data sets to draw conclusions. Helps organizations understand large amounts of data. Steps: Collect raw data → Analyze → Generate insights → Report.

📝Variables vs. Constants

Variable

A named storage location whose value can change during program execution. Like a labeled box you can put different things in. Example: score = 0, then later score = 150.

Constant

A named value that is set once and never changes during execution. Example: PI = 3.14159, MAX_USERS = 100. Makes code more readable and prevents accidental changes.

🔄Control Flow — How Programs Make Decisions & Repeat

If-Else (Branching)

Execute different code based on a condition. If the condition is true, do one thing. Otherwise (else), do another.

While Loop

Execute code as long as a condition remains true. Condition is checked before each iteration. Can loop zero or more times.

For Loop

Execute code a specific number of times. Iterates through a range or collection. Used when you know the count in advance.

Switch/Case

Compare a value against multiple cases. Cleaner than many if-else statements when checking one variable. Example: checking primary color input (red, blue, green).

// If-Else (Branching) IF amount < 5000 THEN interest = 7% ELSE interest = 8% END IF // While Loop — "while the condition is true, keep running" WHILE time < "8:00 PM" monitorNetwork() END WHILE // For Loop with nested loop (COUNTER = 3 × 2 = 6) SET COUNTER = 0 FOR EACH i FROM 1 TO 3 // outer: 3 times FOR EACH j FROM 1 TO 2 // inner: 2 times each COUNTER = COUNTER + 1 END FOR END FOR // Result: COUNTER = 6

📐Functions & Code Reuse

A function is a reusable block of code that performs a specific task. You define it once, then call it many times. Functions are the primary tool for code reuse in programming.

// Defining a function FUNCTION calculateTax(price, rate) tax = price * rate RETURN tax END FUNCTION // Calling (reusing) the function tax1 = calculateTax(100, 0.08) tax2 = calculateTax(250, 0.08)

AI & Emerging Technologies

Artificial Intelligence, IoT, augmented reality, and virtual reality are increasingly tested topics on the Tech+ exam.

🤖Types of AI

Generative AI

Creates new content (text, images, code, audio, video) based on patterns learned from training data. Examples: ChatGPT generating text, DALL-E creating images, AI writing financial reports from data.

Predictive AI

Analyzes patterns to forecast future outcomes. Examples: identifying unusual spending patterns (fraud detection), recommending products, predicting equipment failure.

AI Chatbots

Automated conversational agents that answer questions from a knowledge base. Used for customer service, FAQs, helpdesks. Different from general AI — they are task-specific.

📝

AI Application Examples

Generative AI = Producing automated financial reports from data, generating code. | Predictive AI = Identifying unusual consumer spending (fraud detection). | AI Chatbot = Customer-facing tool answering common questions from a knowledge base.

🌐IoT — Internet of Things

IoT devices are everyday physical objects with internet connectivity and an embedded OS, allowing them to collect and exchange data.

Consumer IoT

Video doorbells, smart thermostats, smart refrigerators, wearables, home assistant speakers, smart light bulbs, IP security cameras.

Industrial IoT

Manufacturing sensors, smart grid systems, agricultural monitoring, medical devices, autonomous vehicles.

🥽AR vs. VR

Augmented Reality (AR)

Overlays digital content onto the real world. You still see your real environment with digital additions on top. Example: furniture preview app — see how a couch looks in your living room before buying. Pokémon Go.

Virtual Reality (VR)

Creates a completely immersive digital environment. You're cut off from the real world. Requires a headset. Example: VR gaming, virtual tours, training simulations.

Troubleshooting Methodology

The CompTIA troubleshooting methodology is a standard 6-step process. Know each step and what it involves — scenario questions will test specific steps.

🔧The 6-Step Troubleshooting Process

1

Identify the Problem

Gather information from the user. Ask questions: "What changed recently? When did it start? Can you reproduce it?" Check for error messages. This is where you gather facts.

2

Establish a Theory of Probable Cause

Based on evidence, form a hypothesis about the most likely cause. Question the obvious. Don't overthink — start simple. "If I see X symptom, the cause is probably Y."

3

Test the Theory to Determine the Cause

Try your solution. If the theory is confirmed, move to step 4. If not, form a new theory (go back to step 2) or escalate. Change and test one thing at a time.

4

Establish a Plan of Action / Implement the Solution

Plan how to fix the problem, considering the impact on users. Implement the fix. Sometimes these are combined as one step: "establish a plan and implement."

5

Verify Full System Functionality

After fixing, confirm the problem is resolved AND that the fix didn't cause new problems. Test the original issue. Get user confirmation. Implement preventive measures.

6

Document Findings, Lessons Learned, Actions & Outcomes

This is the FINAL step. Record what the problem was, what caused it, how it was fixed, and what was learned. Creates a knowledge base for future issues.

⚠️

Common Exam Traps

"What is the FINAL step?" → Document findings. NOT "verify functionality" — that's step 5. | "Before closing the ticket, what must be done first?" → Verify functionality (step 5) before documenting (step 6). | "Asking what recently changed" → That's step 1 (Identify the Problem).

🔍First Step in Identifying Causes

When trying to identify the cause of a problem, the first action should be to check for obvious or common issues — the simple stuff first. Don't start with complex diagnostics when the cable might just be unplugged.
